## 6. Recommandations priorisees

### CRITIQUE (a corriger avant toute nouvelle feature)

1. **`create_json()` et `create_all_tables()` non definies** -- `LoggerSQLExporter.py:188-189` -- Si `new_db=True` (configure dans DataExporterConfig.json), le pipeline crashe immediatement avec `NameError`. Soit importer ces fonctions depuis leur module d'origine, soit retirer le bloc `if new_db:` s'il n'est plus utilise.

### IMPORTANT (a corriger dans les prochains sprints)

1. **`localhost\SQLEXPRESS` en dur dans 3 fichiers** -- `LoggerSQLExporter.py:20`, `kafka_main.py:120,151` -- devrait etre centralise dans `kafka_config.py` comme constante `SQL_SERVER`. Rend le deploiement sur un autre serveur SQL impossible sans modifier 3 fichiers.

2. **`log.error()` sans `exc_info=True`** -- `kafka_producer.py:148` -- dans un bloc `except OSError`, le traceback n'est pas capture. Les 2 autres `log.error()` hors except (lignes 48, 275) sont OK car il n'y a pas d'exception en cours.

3. **Fichiers `.lzma` tracked dans `output/`** -- 3 fichiers binaires toujours suivis par git malgre le `.gitignore`. Necessitent `git rm --cached output/` pour etre retires.

### AMELIORATION (quand tu auras le temps)

1. **Docstrings module** manquantes sur `LoggerPduProcessor.py` et `LoggerSQLExporter.py`.
2. **Lignes > 120 chars** -- 15 occurrences dans `LoggerPduProcessor.py`.
3. **`.gitignore` sans newline final** -- cause des diffs parasites.
4. **Installer `pytest`** et adapter le framework de test pour permettre `pytest --cov`.
5. **`requirements.txt`** -- ajouter des bornes superieures (`<3.0.0`) pour la reproductibilite.
6. **`sys.setrecursionlimit(* 5)`** dans 2 fichiers -- documenter pourquoi c'est necessaire (probablement opendis).
7. **Injection SQL residuelle** dans `legacy/logger.py:364` -- meme pattern corrige dans `kafka_main.py`, mais code legacy.
